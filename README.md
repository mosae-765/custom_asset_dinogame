# 커스텀 공룡 게임

크롬 오프라인 공룡 게임을 본떠 만든 웹 게임입니다. 원하는 PNG 이미지를 직접 넣어서 공룡, 장애물, 땅과 구름 이미지를 원하는 대로 바꿀 수 있습니다.

## 조작법

| 키 | 동작 |
| --- | --- |
| ↑ / Space | 점프 (공중에서 한 번 더 누르면 더블 점프) |
| ↓ | 숙이기 |

모바일에서는 화면 터치로 점프할 수 있습니다. 숙이기는 불가능하지만 게임에 지장이 생기진 않습니다. (더블 점프로 모든 장애물 회피 가능)

## 이미지 렌더링

시작 화면에서 **일반 이미지 / 픽셀 아트** 중 하나를 선택할 수 있습니다.

- **일반 이미지**: 매끄럽게(안티 얼레이싱) 렌더링됩니다.
- **픽셀 아트**: 경계가 또렷하게(계단 형태) 렌더링됩니다. 픽셀 아트 이미지를 사용할 때 선택하세요.

## 점수와 난이도

점수가 오를수록 게임 속도가 점점 빨라지고, 일정 점수마다 화면이 밤/낮으로 전환됩니다. 250점 이후부터는 새(bird) 장애물도 등장합니다. 최고 점수는 브라우저에 저장됩니다.

## 개발자 정보

빌드 도구나 프레임워크 없이 순수 HTML/CSS/JS로만 작성되어 있습니다. 별도 설치 없이 파일을 그대로 열거나 정적 서버로 서빙하면 됩니다.

### 파일 구조

| 파일 | 역할 |
| --- | --- |
| `index.html` | 시작 화면 + 게임 화면 마크업 |
| `style.css` | 전체 스타일 (시작 화면, 게임 화면, 렌더링 모드 토글 등) |
| `assets.js` | `AssetLoader` — 사용자 폴더 / `default_assets/` / 색상 사각형 순으로 이미지를 로드하고 폴백 처리 |
| `game.js` | 게임 루프, 엔티티(`Dino`, `Obstacle`, `Ground`, `Cloud`), 렌더링(`BitmapCache`), 메인 `Game` 클래스 |
| `default_assets/` | 폴더를 선택하지 않았을 때 사용되는 기본 PNG 에셋 |
| `lite/` | 사진 1장만으로 공룡을 커스텀하는 축소 버전 (`assets.js`/`index.html`만 따로 존재, `style.css`는 공유) |

### 좌표/스케일 체계

게임 로직은 `game.js` 상단의 논리 단위 상수를 기준으로 동작합니다.

```js
const CW = 1000;        // 논리 캔버스 너비
const CH = 400;         // 논리 캔버스 높이
const GROUND_Y = 360;   // 땅 표면 y좌표
const SCALE = CH / 300; // 기준 높이(300) 대비 스케일 배수 — 엔티티 크기 계산에 사용
```

모든 엔티티 크기/좌표(공룡, 선인장, 새, 구름, 땅)는 `CW`/`CH`/`SCALE`에서 파생되므로, 이 값들만 바꾸면 전체 비율이 함께 조정됩니다.

실제 화면 표시 크기는 `Game._applyCanvasSize()`에서 `window.innerWidth * 0.7`로 계산되고, `dpr * scale`(`scale = displayW / CW`)을 `this.renderScale`로 저장해 `BitmapCache` 프리스케일 해상도 계산에 사용합니다. 논리 좌표(`CW`×`CH`)와 실제 표시 크기가 분리되어 있어, 창 크기가 바뀌어도 게임 로직은 그대로 유지됩니다.

### 이미지 렌더링 (`BitmapCache`)

`BitmapCache`는 원본 이미지를 `OffscreenCanvas`에 미리 스케일해두고 캐싱합니다.

- **일반 이미지 모드**: 한 번에 크게 축소하지 않고 50%씩 단계적으로 다운스케일(`imageSmoothingQuality: 'high'`)해 화질 저하를 줄입니다.
- **픽셀 아트 모드**: `imageSmoothingEnabled = false`로 단일 단계 변환해 경계를 또렷하게 유지합니다.

캐시 키는 `assetKey|physW|physH` 형태라, 렌더 스케일이 바뀌면(창 크기 변경 등) 새 키로 자동 재생성됩니다.

### 새 장애물/에셋 추가하기

- 새로운 선인장 종류: `CACTUS_CFGS` 배열에 `{ key, baseW, h, count }` 항목을 추가하고 `default_assets/`에 같은 키의 PNG를 넣으면 됩니다.
- 새(bird) 등장 높이: `Game._computeBirdYs()`에서 계산하며, "high" 위치는 사용자가 넣은 `dino_duck1.png` 크기에 맞춰 항상 숙이기로 피할 수 있도록 동적으로 계산됩니다.
- 에셋 키를 추가하려면 `assets.js`의 `ASSET_KEYS`/`DEFAULT_DEFS`에도 함께 등록해야 폴백·사각형 처리가 정상 동작합니다.
